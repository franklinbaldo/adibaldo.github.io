# Diagnóstico UX do Blog Alfarrábios do Adi

**Data:** 28 de março de 2026

## O Que Está Funcionando Bem

1. **Estética Literária Alinhada:** O design transmite perfeitamente a sensação de "pergaminho" e "nostalgia" definida no `DESIGN.md`. A paleta de cores creme/terra e a tipografia (Playfair Display + Source Serif 4) criam um ambiente imersivo para leitura longa.
2. **Desempenho Estático (Astro):** O uso do Astro e SSG (Static Site Generation) garante tempos de carregamento mínimos. A otimização de imagens (via `astro:assets`) é nativa e eficiente.
3. **Modo Leitura Noturna:** A implementação com variáveis CSS suporta de forma nativa e sem flashes (com script em `BaseHead.astro`) o modo noturno.
4. **Livro Imprimível (`/livro`):** Uma feature genial e perfeitamente implementada para o público alvo (leitores de livros físicos e pessoas mais velhas), com estilização focada em impressão (`@media print`).
5. **Integração de Metadados e SEO:** JSON-LD e tags Open Graph estão devidamente implementados, favorecendo o compartilhamento e ranqueamento.

## 1. Avaliação de UX, Performance, Acessibilidade e SEO

### UX & Interface
- **Navegação Consistente:** O uso de *pills* e o menu superior sticky são simples e diretos.
- **Espaçamento e Respiro:** O espaçamento é adequado, priorizando o texto (Astro).
- **Legibilidade de Imagens nas Prosas:** As imagens misturam-se bem com o texto justificado, criando uma autêntica sensação de revista.

### Performance
- Excelente. Como é tudo gerado estaticamente pelo Astro sem JavaScript pesado no client, o Time to Interactive é baixíssimo. O build otimiza imagens automaticamente.

### Acessibilidade (a11y)
- Faltam alguns `aria-labels` claros em ícones do header, especialmente quando a view transiciona para mobile.
- Contrastes parecem razoáveis, mas as *tags* translúcidas no "Featured Article" podem ter problemas de contraste no fundo desfocado de algumas imagens muito claras.
- Foco de navegação no teclado (`:focus-visible`) foi customizado e está bom, assim como o skip-link "Pular para o conteúdo principal".

### SEO
- Uso intensivo de metadados enriquecidos (`schema.org`). As tags canônicas estão em vigor.
- `Pagefind` atua bem na busca mas não adiciona bloat na renderização inicial do site.

## 2. Lista Priorizada de Melhorias

### Quick Wins (Vitórias Rápidas)
1. **Ajuste de Acessibilidade no Featured Card:** Garantir que o overlay de fundo sobre a imagem de destaque seja sempre escuro o suficiente para não comprometer o contraste dos textos e tags (`src/styles/global.css`).
2. **Otimização do Script de View Transitions:** Astro 5 utiliza `ClientRouter`. Certificar-se de que eventos amarrados a navegações, como barras de progresso (`src/layouts/BlogPost.astro`), sejam re-acoplados utilizando `astro:page-load` em vez de apenas contar com carregamentos inteiros de página.
3. **Menu Mobile:** O Header encolhe, mas a navegação ainda ocorre por links horizontais. Avaliar um scroll horizontal mais polido para as *pills* da navegação caso existam muitos links no futuro.
4. **Melhoria da Paginação:** Incluir indicador numérico além dos botões "Textos mais antigos/recentes" se o número de postagens crescer de maneira drástica. O texto "Anterior" para indicar postagens antigas e "Próximo" para as recentes está ok, mas pode gerar ambiguidade.

### Melhorias Complexas / Longo Prazo
1. **Página de Categorias/Tags mais Elaborada:** O design sugere um "índice de categorias como livro antigo". As tags atuais são *pills*. Seria interessante criar uma view `/indice` remetendo ao sumário do Livro Imprimível.
2. **Integração de Galeria de Imagens Estilizada:** O design cita "Álbum de fotos de lugares marcantes" (ainda não presente). Criar um layout `Album.astro` específico focado em fotografia.
3. **Página de Locais (Mapa):** Os locais já existem (e há referência de link-lo ao layout de `/locais`), porém falta um mapa rudimentar ou visual (mesmo em imagem clicável) que reforce a história regional em Rondônia/Brasil.

## 3. Features Astro para Agregar Valor

1. **Astro Content Collections API (Avançado):** Utilizar `reference()` para criar relações fortes entre Postagens -> Autores ou Postagens -> Locais, tornando os dados mais consistentes e as páginas genéricas mais ricas.
2. **Astro Image (Responsive Art Direction):** O componente `<picture>` nativo pode servir imagens diferentes dependendo do *device* (desktop vs mobile). Seria útil para imagens Hero no Featured Post onde partes importantes da foto acabam sendo cortadas.
3. **Astro View Transitions API aprofundada:** Como o blog simula um livro de alfarrábios, adicionar animações de *fade* cruzado suave entre páginas (`transition:animate="fade"`) pode reforçar muito o tom "literário" em oposição aos cortes bruscos da web.
